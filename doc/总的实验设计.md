## 问题一：以"云调度"视角介绍四种 MARL 算法

### 核心框架：CTDE 与调度层次结构的映射

在云平台弹性资源调度中，存在天然的**两级调度层次**：
- **上层（服务器群选择）**：Server_Farm Agent → `Discrete(num_farms)`
- **下层（服务器选择）**：Server Agent → `Discrete(num_servers_in_farm)`

四种 MARL 算法恰好代表了四种不同的**多智能体协作假设**，每种假设对应云调度中不同的信息共享和协调机制。

---

### 各算法的调度语义介绍框架

#### IDQN —— "孤岛式自治调度"
- **调度映射**：现实中各数据中心/子集群各自为政，没有跨节点协调机制
- **信息假设**：每个调度智能体仅观测本地资源状态，不感知其他智能体的决策
- **局限性（调度视角）**：Farm Agent 选择了某个农场，但 Server Agent 对此一无所知，导致两级决策脱耦，可能出现"Farm 选了过载的集群，Server 无法调度"的协调失效
- **介绍重点**：作为分布式调度的 baseline，体现**无协调代价**

#### VDN —— "线性贡献假设下的协同调度"
- **调度映射**：假设整体调度质量（总能耗 + 任务完成率）等于各级智能体贡献的**线性叠加**
- **调度含义**：Farm 级和 Server 级的调度决策对整体能耗的影响是独立可加的
- **局限（调度视角）**：现实中农场选择会限制可用服务器集合，两级决策存在强耦合——VDN 的线性分解无法捕捉这种**选择-约束级联效应**
- **介绍重点**：体现**层次调度耦合性**问题

#### QMIX —— "单调约束下的非线性协同调度"
- **调度映射**：允许两级调度决策的非线性交互，同时通过单调性约束保证"局部最优 ⟺ 全局最优"——即每个调度智能体的局部贪心可以被证明等价于全局最优联合调度
- **架构体现**：Mixer 网络由全局状态（所有农场、所有服务器利用率）生成权重，直接对应云平台的**全局资源视图**
- **调度优势**：在异构环境中，Mixer 可以学习到"将 CPU 密集型任务路由至高效率服务器"的非线性协作模式
- **介绍重点**：**分布式执行 + 集中式全局资源优化**，对应云平台"全局调度器指导本地执行"范式

#### MAPPO —— "集中式资源视图驱动的策略梯度调度"
- **调度映射**：Centralized Critic 直接观测全数据中心状态（所有农场 CPU 利用率、效率层级），等价于**云管理平台的全局监控视图**；各 Agent 只执行本地策略，等价于**分布式调度器**
- **调度优势**：策略梯度天然支持**连续优化信号**，Critic 的全局视图使其能感知跨层调度的能耗-时延 trade-off
- **与 QMIX 的差异（调度视角）**：QMIX 是值函数分解（离散动作高效），MAPPO 是策略梯度（稳定性更好，适合在线部署场景）
- **介绍重点**：体现**在线弹性调度**与**全局能耗感知**的结合

### 需要新增的指标（需代码实现）

#### 1. 资源利用均衡性指标 —— Jain's Fairness Index
```
J = (Σ u_i)² / (n × Σ u_i²)   其中 u_i 为第 i 台服务器 CPU 利用率
J ∈ [0,1]，越大越均衡
```
**体现**：MARL 算法是否实现更均匀的负载分布（防止热点）

#### 2. 异构感知利用率 —— Heterogeneity Exploitation Rate (HER)
```
HER = Σ(task_cpu_i × efficiency_tier_j) / Σ(task_cpu_i)
其中 j 为分配给任务 i 的服务器
```
**体现**：算法是否学会将 CPU 密集型任务优先路由至高效率服务器（创新点核心指标）

#### 3. 弹性指标 —— 活跃服务器比率 (Active Server Ratio)
```
ASR = num_active_servers / total_servers  (越小越好，更好的资源整合)
```
**体现**：弹性调度能力——是否实现了资源的动态整合与闲置关闭

#### 4. 单任务能耗 —— Energy Efficiency per Task (EET)
```
EET = total_energy_cost / num_completed_tasks
```
**体现**：标准化能耗，排除任务数量差异的干扰

#### 5. SLA 违约率 —— Deadline Miss Rate (DMR)
```
DMR = tasks_exceeding_deadline / total_tasks
```
注：需要确认现有 `task.deadline` 字段是否在评估时被追踪

### 推荐实验设计

#### 实验一：全算法横向对比（主实验）
**目标**：回答"哪种调度策略在云平台资源调度中综合表现最优"

| 对比维度 | 指标 | 意义 |
|---------|------|------|
| 调度质量 | 任务完成率、拒绝率 | 基础调度能力 |
| 能耗效率 | 总能耗、EET | 弹性节能目标 |
| 资源均衡 | Jain's Fairness Index | 负载均衡 |
| 异构感知 | HER | 本文创新点 |
| 算法收敛 | 奖励曲线 + 收敛轮次 | 学习效率 |

**方法**：9 个算法 × 5 次随机种子 → 报告均值 ± 标准差

---

#### 实验二：异构环境消融实验（创新点实验）
**目标**：证明"MARL 算法在 CPU 异构场景下的适应性"是本文贡献

**设计**：
- 变量：服务器混合比例 `balanced / legacy / modern` (代码已有)
- 变量：异构奖励开关 `use_heterogeneity = True / False` (代码已有)
- 算法：重点对比 QMIX、MAPPO vs IDQN、传统基线

| 实验条件 | 期望结论 |
|---------|---------|
| balanced + hetero=True | QMIX/MAPPO 的 HER 显著高于 IDQN 和传统基线 |
| legacy + hetero=True | 所有算法能耗上升，MARL 算法退化更慢 |
| modern + hetero=True | 新一代服务器占多数，MARL 节能收益最大 |
| balanced + hetero=False | HER 无差异，验证异构奖励设计的有效性 |

**核心结论支撑**：MARL 算法（特别是有 centralized critic 的 QMIX/MAPPO）能学习到异构感知路由策略

---

#### 实验三：调度收敛与稳定性分析
**目标**：比较 MARL 算法的训练效率和策略稳定性

**指标**：
- 达到 baseline 最优（Energy-Greedy）性能所需的训练轮次
- 收敛后奖励的方差（策略稳定性）
- 离策略（QMIX/VDN/MADDPG）vs 在策略（MAPPO）的样本效率

---

#### 实验四：弹性扩展实验（已有 exp3_scalability.py 框架）
**目标**：验证 MARL 在不同规模云集群的适用性

**增强现有实验**：
- 在不同规模下记录 ASR（活跃服务器比率）
- 对比 MARL 和传统算法在规模增大时的性能衰减曲线

---

### 指标与研究结论的对应关系

| 研究结论 | 支撑指标 | 对应实验 |
|---------|---------|---------|
| MARL 优于传统调度算法 | 任务完成率、总能耗、EET | 实验一 |
| CTDE 算法优于独立学习 | 奖励曲线、HER、Jain's J | 实验一、三 |
| 异构感知路由是有效创新点 | HER、能耗 × server_mix | 实验二 |
| QMIX/MAPPO 的全局视图带来调度优势 | HER、Jain's J、能耗标准差 | 实验二 |
| MARL 具备弹性扩展能力 | ASR、规模扩展能耗增长率 | 实验四 |
