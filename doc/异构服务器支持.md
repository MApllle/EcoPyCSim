# 异构服务器支持 — EcoPyCSim 改进计划

## Context

当前代码中所有服务器同质（alpha 随机均匀分布，beta 固定为 10，Opt 硬编码为 0.7），无法反映真实数据中心的异构性。改进目标：引入 old/mid/new 三代服务器、将能效等级暴露给 agent 观测、在 reward 中加入异构感知项，并预留消融实验控制选项。

## 功率验证（80% 利用率时 old/new 差距）

| 代际 | α    | β  | Opt | 动态功率@80% util                       |
|------|------|----|-----|----------------------------------------|
| old  | 0.75 | 13 | 0.6 | 0.6×0.75 + (0.8−0.6)²×13 = **0.97** |
| mid  | 0.50 | 10 | 0.7 | 0.7×0.50 + (0.8−0.7)²×10 = **0.45** |
| new  | 0.30 | 7  | 0.8 | 0.8×0.30 + 0              = **0.24** |

old/new 比值 ≈ 4×，满足"3-5×"要求。

### 1. `components/model_scripts/make_server_farms.py`

**a) 在 import 后、第一个函数定义前加入模块级常量：**

```python
SERVER_GENERATIONS = {
    "old": {"alpha": 0.75, "beta": 13, "opt": 0.6,  "efficiency_tier": 0.0},
    "mid": {"alpha": 0.50, "beta": 10, "opt": 0.7,  "efficiency_tier": 0.5},
    "new": {"alpha": 0.30, "beta":  7, "opt": 0.8,  "efficiency_tier": 1.0},
}

PROPORTION_PRESETS = {
    "balanced": {"old": 0.33, "mid": 0.34, "new": 0.33},
    "legacy":   {"old": 0.60, "mid": 0.30, "new": 0.10},
    "modern":   {"old": 0.10, "mid": 0.20, "new": 0.70},
}
```

**b) 新增辅助函数（加在 `add_graph_vertices_and_edges_attributes` 前）：**

```python
def _assign_generation_labels(n_servers, server_proportions):
    counts = {}
    assigned = 0
    gens = ["old", "mid", "new"]
    for i, gen in enumerate(gens):
        if i < len(gens) - 1:
            counts[gen] = round(server_proportions.get(gen, 0.0) * n_servers)
            assigned += counts[gen]
        else:
            counts[gen] = n_servers - assigned  # 最后一类吸收舍入误差
    labels = []
    for gen in gens:
        labels.extend([gen] * counts[gen])
    return labels
```

**c) 修改 `add_graph_vertices_and_edges_attributes` 签名：**
```python
def add_graph_vertices_and_edges_attributes(g, seed=None, server_proportions=None):
```

在函数体的顶层循环前加入两条路径（`server_proportions is None` 时保持原有随机 alpha 逻辑，否则走代际分配）。循环需改为 `for i, vertex in enumerate(new_graph.vs):`，并为每个 vertex 额外写入：
```python
vertex['Server_Generation'] = gen_name
vertex['Opt_Utilization']   = gen_params["opt"]
vertex['Efficiency_Tier']   = gen_params["efficiency_tier"]
```
`server_proportions is None` 时三个新属性分别填 `'mid'`, `0.7`, `0.5`（保持向后兼容）。

**d) 修改 `create_a_server_farm` 和 `create_server_farms` 签名：**
均加入 `server_proportions=None` 参数，透传下去。

### 5. `env/cloud_scheduling.py`

**a) `__init__` 加入三个新参数：**
```python
def __init__(self, num_jobs, num_server_farms, num_servers, render_mode=None,
             use_heterogeneity: bool = True,
             hetero_weight: float = 0.3,
             server_proportions=None):
```
body 中加入：
```python
self.use_heterogeneity  = use_heterogeneity
self.hetero_weight      = hetero_weight
self.server_proportions = server_proportions
self._last_hetero_reward = 0.0
```

**b) `reset()` 透传 `server_proportions`：**
```python
server_farms = initialize_server_farms(
    self.num_servers, self.num_server_farms, seed,
    server_proportions=self.server_proportions
)
```
并加入 `self._last_hetero_reward = 0.0`。

**c) `observation_space`：**

两种 agent 的 `spaces.Dict` 都新增：
```python
"efficiency_tiers": spaces.Box(low=0, high=1.0, shape=shape, dtype=float),
```
shape 与对应的 `cpus_utilization` 完全相同。

**d) `_get_observation`：**

server_farm agent：
```python
eff_tiers = np.array([self.server_farms[key].efficiency_tiers for key in self.server_farms.keys()])
obs["efficiency_tiers"] = eff_tiers
```

server agent：
```python
eff_tiers = np.array(server_farm.efficiency_tiers)
obs["efficiency_tiers"] = eff_tiers
```

**e) `_get_reward` — 新增异构感知项（两个 agent 共享同一 bonus）：**
```python
hetero_bonus = 0.0
if self.use_heterogeneity and not self.task_rejected_status:
    selected_server = self.server_farms[self.server_farm_id].servers[str(self.server_id)]
    hetero_bonus = (
        selected_server.efficiency_tier
        * self.scheduled_task_cpu
        * self.hetero_weight
    )
    self._last_hetero_reward = hetero_bonus

if agent == "server_farm":
    combined_reward = energy_saving_reward + (task_success_reward * 0.5) + hetero_bonus
elif agent == "server":
    combined_reward = task_success_reward + (energy_saving_reward * 0.5) + hetero_bonus
```

**f) `info()` 中 server_farm agent 加入：**
```python
"hetero_reward_total": self._last_hetero_reward,
```

### 6. `main.py`（改写为消融实验脚本）

结构：
- 顶部 `CONFIG` dict 控制 `num_jobs / num_server_farms / num_servers`
- `run_episode(env)` 函数运行一个完整 episode，返回指标 dict（total_energy_cost, rejection_rate, hetero_reward_total, steps）
- 主循环遍历 `3 比例配置 × 2 异构开关 = 6 组合`，复用 `PROPORTION_PRESETS`
- 最后打印对齐表格

## 向后兼容性

- `server_proportions=None` 时，`make_server_farms.py` 走原有随机 alpha 路径，`run_env.py` 零改动可运行
- `Server.__init__` 所有新参数均有默认值
- `CloudSchedulingEnv` 所有新参数均有默认值

## 验证方式

1. `python run_env.py` — 应正常运行，无任何错误（向后兼容验证）
2. `python main.py` — 输出 6 组消融结果表格；modern+hetero_on 组预期 hetero_reward_total 最高
3. 对比 `legacy` vs `modern` 配置的 total_energy_cost：modern 应显著更低（新一代服务器更省电）
4. `hetero_on` vs `hetero_off` 在相同比例配置下：rejection_rate 应相当，但 hetero_on 的 reward 组成不同
